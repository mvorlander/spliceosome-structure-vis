Dashboard Documentation

features, UI, and curation methodology

Scope

This lightweight dashboard lists deposited spliceosome and spliceosomal snRNP structures and provides ChimeraX scripts that download models directly from the PDB.

  • No mmCIF files, local curated CIF entries, or large map binaries are bundled here.
  • Primary EMDB maps can be downloaded on demand by using the primary-map ChimeraX script variant.
  • The full internal resource adds shared coordinate files, renamed-chain mmCIFs, all-map support, and local curated structures.
  • Entries are organized by pathway state, species, publication, resolved components, RNA annotations, quality metrics, and color/homolog metadata.

Structure UI

The Structures tab is the main workspace for finding entries and copying ChimeraX scripts.

  • Use the sidebar to filter by text, complex, species, PRPF8/SF3B1 content, RNA features, and sort order.
  • Click a state in the splicing-cycle cartoon to filter the structure table.
  • Select a row to open details with paper metadata, components, RNA annotations, quality metrics, curation audit summaries, and a script preview.
  • Use the Compare column to keep up to five structures in a temporary comparison panel.

RNA Features

Resolved substrate and snRNA feature annotations are generated from deposited sequences and coordinates.

  • The RNA feature text box narrows the feature drop-down; choosing a feature filters the structure table.
  • Common presets expose substrate annotations, snRNA annotations, 3' exon, ligated exons, branch A, and U6 ACAGAGA.
  • Named ChimeraX selectors follow `stage_PDB_feature`, for example `pre_Bact_7AAV_5exon`.
  • When multiple chains or segments represent one biological feature, the script creates a single named selection that unions the residue specs.

Protein Colors And Homologs

The Protein Colors tab exposes the universal color table, common aliases, homolog mappings, and source/provenance metadata.

  • Colors combine provided ChimeraX examples, curated palettes, and generated class/family palettes.
  • This public dashboard uses original deposited chain IDs; renamed-chain files and canonical-chain script variants are intentionally omitted.
  • Homolog mapping combines deposited names, curated aliases, SpliceosomeDB-style classes, OMA/BLAST-supported mappings, and manual corrections.

ChimeraX Scripts

Public scripts use original chain IDs and `open PDBID` commands so ChimeraX retrieves models directly from the PDB.

  • Scripts use unique model IDs so multiple structures can be opened in one ChimeraX session.
  • Structures are aligned to a shared anchor frame where PRPF8/SF3B1-compatible anchors are available.
  • Scripts apply the shared color code, RNA feature named selections, default display style, and nucleic-acid visibility.
  • The model-only variant loads coordinates only; the primary-map variant also downloads the deposited primary EMDB map and brings it with the aligned model where ChimeraX can move the opened volume model.

Methods-Style Curation Methodology

The resource is generated by local scripts that combine public structural metadata, deposited coordinate files, EMDB records, curated spliceosome knowledge, and project-specific validation tables. The following subsections describe the procedures in a form intended to be adaptable for a scientific Methods section. For publication use, the exact metadata snapshot, script commit, query log, and any manual overrides should be reported together with the exported tables.

Structure Discovery And Inclusion Criteria

Candidate deposited entries were identified from PDBe/PDB query sets recorded in `human_spliceosome_cryoem/metadata/all_species/discovery_log.csv`, with retained entries and discovery queries listed in `structures.csv` and excluded or failed records listed in `skipped_entries.csv`. Discovery was seeded with PRPF8/Prp8 and SF3B1/Hsh155-containing spliceosome entries, then expanded by title, composition, publication context, SpliceosomeDB-style seeds, and curated review to include spliceosomal snRNP assemblies, model-organism spliceosomes, and E-complex entries lacking those anchors. Deposited cryo-EM entries were retained when the deposited title, polymer composition, publication context, or curated inspection supported assignment to a spliceosome pathway bin or spliceosomal snRNP state. The public dataset comprises released PDB entries only; the full internal dashboard additionally includes local curated CIF-only examples, which should be excluded from deposited-entry counts unless stated explicitly.

Primary Metadata Extraction

Entry-level metadata were taken from PDBe/PDB entry and publication records and cached in `entries_raw.json`. Titles, release and revision dates, experimental methods, nominal resolutions, linked EMDB identifiers, publication titles, DOIs, PubMed identifiers, journals, years, organisms, and deposited polymer entities are written to `structures.csv` and `publications.csv`. Polymer entities, chains, sequences, molecule names, gene names, polymer types, and UniProt cross-references were parsed from deposited mmCIF files and PDBe entity records into `components.csv` and `entity_xrefs.csv`. EMDB and validation-derived fields are written to `em_maps.csv`, `map_provenance.csv`, and `quality_metrics.csv` when available.

Splicing-Pathway Assignment

Structures were assigned to curated pathway bins using ordered title- and publication-derived rules implemented in `classify_structure()`, followed by manual correction of known ambiguous cases. The pathway categories include isolated snRNP states, E complex, A complex, tri-snRNP, pre-B, B, pre-Bact, Bact, B*, C, C*, P, post-P/Step2, ILS/disassembly, DIS, and U5 recycling. Ambiguous title patterns such as pre-B versus P complex are flagged in audit outputs such as `audits/stage_assignment_suspects.csv` and corrected where supported by publication or component evidence. Pathway labels should therefore be interpreted as curated bins rather than depositor-supplied stage labels.

Protein And RNA Component Annotation

Protein and RNA components were compiled from deposited polymer entities. Deposited entity names and gene symbols were used as initial labels; UniProt cross-references, curated aliases, species-specific homolog tables, paper-text evidence, OMA/BLAST-supported assignments for model organisms, and manual corrections were then used to normalize labels for search and coloring. Conflicts, missing assignments, and manual overrides are recorded in audit tables and should be treated as curation decisions rather than primary deposition metadata. RNA identities are assigned from deposited molecule or gene names, chain conventions where informative, and sequence-derived evidence. Unrecognized or ambiguous RNA entities are retained as unresolved and exposed in RNA identity audit tables.

Homolog Mapping

Homologous proteins across human, S. cerevisiae, S. pombe, C. elegans, and Chaetomium thermophilum were mapped to human-centric color and class keys where possible. Homolog assignments are evidence-integrated rather than BLAST-only: support may come from self or species matches, curated aliases, paper-text evidence, OMA orthology, BLAST best hits that pass manual review thresholds, and name or color-key fallbacks. Ambiguous or unmatched components remain flagged in unresolved-component and homolog-resolution audits. Repeated Sm/LSm copy labels use snRNP context plus nearest-snRNA heuristics where deposited identity is insufficient.

RNA Feature Annotation

Resolved substrate and snRNA feature annotations were generated by method-specific rules, including deposited component names, author residue numbering, sequence motifs, splice-site inference, resolved-residue checks against the mmCIF atom-site table, and review-derived approximate intervals for conserved snRNA regions. Substrate features include the 5' exon, 5' splice site, intron, branch point region, branch point adenosine, polypyrimidine tract, 3' splice site, 3' exon, ligated exons, intron lariat, and whole resolved substrate. snRNA features include conserved functional regions such as U1 5' splice-site recognition elements, U2 branchpoint-pairing and U2/U6 helix regions, U4/U6 interaction regions, U5 loop I and exon-contact sides, U6 ACAGAGA box, U6 catalytic and helix regions, and Sm/LSm-site regions. Each feature carries method and confidence fields in `substrate_features.csv` or `snrna_features.csv`; review-region or low/medium-confidence annotations should be interpreted as approximate selectable coordinate intervals. A missing feature means the scanner did not find a resolved selectable interval under the current rules, not that the biological feature is absent from the construct or complex. Selector names follow `stage_PDB_feature`; chain names are excluded, and multiple chains or segments representing the same feature are combined into a single named selection.

EM Map Enumeration And File Availability

Primary and associated EMDB records are collected in the full resource from PDB and EMDB cross-references, including focused refinements, linked additional maps, and half maps where listed. Map provenance is tracked separately from map file availability in `em_maps.csv` and `map_provenance.csv`. The full resource records expected map paths and whether each map is present on shared storage; large EM maps are intended to reside on the shared group volume rather than local OneDrive storage. This public lightweight dashboard does not distribute map binaries, but primary-map script variants can request the deposited primary EMDB map directly from ChimeraX.

ChimeraX Script Generation And Alignment

Public scripts retain original deposited chain IDs and use ChimeraX `open PDBID` commands. Model IDs are assigned uniquely per structure to avoid clashes when multiple scripts are run in one ChimeraX session. Structures are aligned into a common reference frame using PRPF8 when available, or SF3B1-compatible anchors where appropriate. Scripts apply a shared display style, color code, named subcomplex selections, RNA feature selections, and nucleic-acid visibility. The full internal resource additionally generates renamed-chain script variants and all-associated-map loading scripts.

Color Assignment

Protein and RNA colors are visualization conventions from a merged universal color table, not evidence for component identity. Colors inherited from provided ChimeraX example scripts were preserved where available, and missing entries were assigned using class/family-aware palettes so related spliceosomal modules retain visually coherent colors. Color provenance distinguishes inherited, curated, and generated colors.

Quality Metrics And Validation Data

Nominal resolutions, model-validation summaries, map metadata, particle counts, detector/microscope information, and Q-score values are extracted from PDBe/EMDB validation and experiment metadata when available. Missing values are reported as unavailable rather than inferred. Validation metrics are displayed in the selected-structure details view to avoid overloading the main table. These fields are screening aids and do not replace inspection of validation reports, local resolution, model-map fit, and deposited maps.

Audit Tables And Confidence Labels

Generated audit tables track curation provenance, assignment confidence, data completeness, map provenance, homolog resolution, RNA identity, unresolved components, and repeated substrate-RNA consistency. These audits support internal review and are summarized in the dashboard through high, medium, review, flags, RNA-annotation, and map-status badges. Confidence labels are rule-based curation states, not calibrated probabilities or independent validation metrics. High confidence denotes deposited, self, manual, or independently supported assignments; medium confidence denotes rule-based or single-source inferred assignments; review denotes missing, conflicting, low-quality, or heuristic assignments requiring manual inspection. Entries marked medium or review are usable for browsing but should be checked before quantitative or publication-quality interpretation.
